Programación desde Cero con PSeInt

Esta guía está pensada para personas que nunca han programado. Incluye ejemplos, errores intencionales y correcciones. El objetivo no es memorizar, sino aprender a pensar como programador.

Enfoque: lógica + hábitos Herramienta: PSeInt Lenguaje: Pseudocódigo en español Incluye: tabla de errores + ejercicios

0) ¿Qué es programar?

Programar es dar instrucciones a una computadora para que haga algo. Un programa es una secuencia de pasos que transforman datos en resultados. PSeInt te permite escribir esos pasos en pseudocódigo (español estructurado) para aprender lógica de programación sin preocuparte por un lenguaje complejo.

Lo que siempre existe en un programa

1) Datos (edad, nombre, puntaje).
2) Decisiones (si aprobó, si es mayor de edad).
3) Repeticiones (contar, sumar, recorrer listas).
4) Acciones (mostrar, leer, calcular).

Cómo piensa un programador

Divide un problema grande en partes pequeñas: "pedir datos al usuario", "hacer el cálculo", "mostrar el resultado". Cada parte se vuelve código.

¿Qué es PSeInt?

PSeInt (Pseudo Intérprete) es una herramienta educativa que te permite escribir algoritmos en pseudocódigo en español y ejecutarlos. También genera diagramas de flujo automáticamente.

¿Por qué pseudocódigo?

Porque te enseña la lógica sin distraerte con reglas complicadas de un lenguaje real. Una vez que domines la lógica, migrar a Python, Java, C++ o cualquier otro lenguaje será mucho más fácil.

✓
Meta de esta guía
Que el estudiante pueda leer un error, ubicarlo en el código y corregirlo con seguridad.

0.1) La interfaz de PSeInt

Antes de escribir tu primer algoritmo, necesitas conocer la herramienta. PSeInt tiene una interfaz sencilla pero completa. Aquí te explico cada parte para que sepas exactamente dónde escribir, cómo ejecutar y dónde ver los resultados.

✓
¿Cómo consigo PSeInt?
Descárgalo gratis desde pseint.sourceforge.net. Está disponible para Windows, Linux y Mac. No necesita instalación compleja: lo descargas, lo abres y ya puedes escribir código.

La consola de ejecución

Cuando presionas ▶ Ejecutar (o F9), se abre una ventana negra (la consola). Ahí sucede todo:

• Lo que escribas con Escribir aparece como texto en la consola.
• Cuando el programa llega a un Leer, la consola se pausa y espera que el usuario escriba algo y presione Enter.
• Al terminar, aparece el mensaje "Presione cualquier tecla para continuar..."

El diagrama de flujo

PSeInt puede convertir tu pseudocódigo en un diagrama de flujo automáticamente (botón o F7). Es una representación visual de la lógica de tu programa:

• Rectángulos: instrucciones normales (asignaciones, Escribir).
• Rombos: decisiones (Si/Entonces, condiciones de ciclos).
• Paralelogramos: entrada/salida de datos (Leer, Escribir).
• Flechas: conectan los bloques y muestran el flujo de ejecución.

!
Aprovéchalo
El diagrama de flujo es una de las mayores ventajas de PSeInt. Si no entiendes cómo funciona tu código, dibuja el diagrama y sigue las flechas. Es como un mapa de tu programa.

Tu flujo de trabajo en PSeInt (paso a paso)

Cada vez que trabajes en un ejercicio, sigue estos pasos:

1. Abre PSeInt y crea un nuevo archivo (Ctrl + N).
2. Cambia el nombre del algoritmo: reemplaza sin_titulo por algo descriptivo (ej: CalcularPromedio).
3. Escribe tu código dentro del bloque Algoritmo ... FinAlgoritmo.
4. Revisa el panel de errores (abajo). Si hay errores, corrígelos antes de ejecutar.
5. Presiona ▶ Ejecutar (F9) para probar.
6. Si algo no funciona como esperas, usa Ejecutar paso a paso (F5) para ver qué pasa en cada línea.
7. Cuando funcione, guarda tu archivo (Ctrl + G).
8. Opcional: genera el diagrama de flujo (F7) para visualizar tu lógica.

1) Variables y Entrada/Salida

Una variable es una "caja con nombre" donde guardas un valor. Ese valor puede cambiar. En PSeInt usas Definir para crear una variable, <- para asignarle un valor, Escribir para mostrar datos y Leer para pedir datos al usuario.

Declarar y asignar variables

Declarar es crear la variable y decirle qué tipo de dato guardará. Luego le asignas un valor con <-.

Ejemplo: declarar y asignar
Algoritmo MisVariables
	Definir vidas Como Entero;
	Definir nombre Como Cadena;
	Definir velocidad Como Real;

	vidas <- 3;
	nombre <- "Mario";
	velocidad <- 120.5;

	Escribir "Vidas: ", vidas;
	Escribir "Nombre: ", nombre;
	Escribir "Velocidad: ", velocidad;
FinAlgoritmo

Cómo nombrar una variable

✅ Descriptiva: velocidad_jugador
✅ Sin espacios: usa _ para separar palabras
✅ Sin tildes ni ñ en el nombre
❌ No empezar con número: 2vidas
❌ No usar palabras reservadas: Si, Para, Mientras

Escribir (mostrar en pantalla)

Mostrar datos al usuario
Algoritmo Salida
	Definir edad Como Entero;
	edad <- 20;

	// Texto fijo
	Escribir "Hola mundo";

	// Texto + variable (con coma)
	Escribir "Tu edad es: ", edad;

	// Varias variables juntas
	Escribir "Edad: ", edad, " años";
FinAlgoritmo

Leer (pedir datos al usuario)

Recibir datos del usuario
Algoritmo Entrada
	Definir nombre Como Cadena;
	Definir edad Como Entero;

	Escribir "¿Cómo te llamas?";
	Leer nombre;

	Escribir "¿Cuántos años tienes?";
	Leer edad;

	Escribir "Hola ", nombre, ", tienes ", edad, " años";
FinAlgoritmo
!
Importante: el operador de asignación
En PSeInt, para guardar un valor en una variable se usa <- (flecha), no =.
✅ vidas <- 3; → Correcto: guarda 3 en vidas.
❌ vidas = 3; → Incorrecto para asignar (el = se usa para comparar).

2) Estructura del algoritmo

Todo programa en PSeInt tiene una estructura clara: empieza con Algoritmo NombreDelAlgoritmo y termina con FinAlgoritmo. Cada bloque (condicional, ciclo, etc.) se abre y se cierra con palabras clave específicas como FinSi, FinPara, FinMientras.

Estructura mínima

Plantilla base de todo programa
Algoritmo MiPrograma
	// Aquí van las instrucciones
	Escribir "Hola mundo";
FinAlgoritmo

Todo lo que esté entre Algoritmo y FinAlgoritmo se ejecuta.

Bloques y sus cierres

Cada estructura tiene su palabra de cierre:

Si ... Entonces → FinSi
Para ... → FinPara
Mientras ... Hacer → FinMientras
Repetir ... → Hasta Que
Segun ... Hacer → FinSegun
SubProceso ... → FinSubProceso

Indentación (buena práctica)

Aunque PSeInt no obliga a indentar (a diferencia de Python), la indentación hace que tu código sea mucho más fácil de leer. Usa tabulaciones para marcar qué código está "dentro" de un bloque.

✅ Con indentación (legible)
Algoritmo Ejemplo
	Definir vidas Como Entero;
	vidas <- 3;

	Si vidas > 0 Entonces
		Escribir "Sigues vivo";
	SiNo
		Escribir "Game Over";
	FinSi
FinAlgoritmo

❌ Sin indentación (difícil de leer)

El mismo código, sin indentar
Algoritmo Ejemplo
Definir vidas Como Entero;
vidas <- 3;
Si vidas > 0 Entonces
Escribir "Sigues vivo";
SiNo
Escribir "Game Over";
FinSi
FinAlgoritmo
✕
¿Funciona? Sí. ¿Es legible? No.
Sin indentación es muy difícil ver qué código pertenece al Si y qué al SiNo.
!
Comentarios
Usa // para escribir comentarios. PSeInt los ignora al ejecutar, pero te ayudan a explicar tu código:
// Esto es un comentario
Checklist de estructura (para que revises solo) hábito
  • ¿Mi código empieza con Algoritmo NombreDelAlgoritmo?
  • ¿Mi código termina con FinAlgoritmo?
  • ¿Cada Si tiene su FinSi?
  • ¿Cada Para tiene su FinPara?
  • ¿Cada Mientras tiene su FinMientras?
  • ¿Estoy indentando el contenido de cada bloque?
  • ¿Cada instrucción ejecutable termina con ; (punto y coma)?

3) Tipos de datos

Un "tipo de dato" es qué clase de cosa guarda una variable. No es lo mismo guardar un número que un texto. Si mezclas tipos sin querer, aparecen errores. En PSeInt hay 5 tipos básicos.

Los 5 tipos de PSeInt

Tipo Qué guarda Ejemplos Uso típico
Entero Números sin decimales 0, 1, -5, 100 Edades, contadores, vidas, puntaje
Real Números con decimales 3.14, -0.5, 120.0 Promedios, precios, porcentajes
Cadena Texto (entre comillas) "Hola", "Mario" Nombres, mensajes, respuestas
Logico Verdadero o Falso Verdadero, Falso Estados, banderas (¿está vivo?)
Caracter Un solo carácter "A", "z", "5" Opciones de menú, letras sueltas

Ejemplo con todos los tipos

Declaración de cada tipo
Algoritmo TiposDeDatos
	Definir vidas Como Entero;
	Definir promedio Como Real;
	Definir nombre Como Cadena;
	Definir esta_vivo Como Logico;
	Definir inicial Como Caracter;

	vidas <- 3;
	promedio <- 8.5;
	nombre <- "Mario";
	esta_vivo <- Verdadero;
	inicial <- "M";

	Escribir "Nombre: ", nombre;
	Escribir "Vidas: ", vidas;
	Escribir "Promedio: ", promedio;
	Escribir "¿Vivo? ", esta_vivo;
	Escribir "Inicial: ", inicial;
FinAlgoritmo
!
Declarar varias variables del mismo tipo
Puedes declarar varias en una sola línea:
Definir x, y, z Como Entero
Definir nombre, apellido Como Cadena
¿Por qué importa el tipo? concepto
Porque determina qué operaciones puedes hacer:
  • Puedes sumar números: 3 + 2 → 5
  • Puedes unir textos: "Hola" + " Mundo" → "Hola Mundo"
  • Pero "Hola" + 3 puede dar error o resultados inesperados si no conviertes.
  • Para convertir número a texto: ConvertirATexto(3)
  • Para convertir texto a número: ConvertirANumero("42")

4) Operadores (matemáticos, comparación y lógica)

Los operadores son los "verbos" de la programación: le dicen al programa qué hacer con los datos. Sin operadores no podrías sumar puntaje, comparar notas, ni decidir si un alumno aprueba.

4.1 · Operadores matemáticos (aritméticos)

Operador Nombre Ejemplo Resultado ¿Para qué sirve?
+ Suma 10 + 3 13 Sumar puntaje, acumular totales
- Resta 10 - 3 7 Calcular diferencias, restar vidas
* Multiplicación 10 * 3 30 Calcular áreas, multiplicar precios
/ División 10 / 3 3.33... Promedios, repartir cantidades
MOD Módulo (residuo) 10 MOD 3 1 Saber si es par/impar, ciclos de turnos
^ Potencia 2 ^ 3 8 Cuadrados, cubos, interés compuesto
Ejemplo completo: cálculos básicos
Algoritmo OperadoresMatematicos
	Definir a, b Como Entero;
	a <- 10;
	b <- 3;

	Escribir "Suma: ", a + b;
	Escribir "Resta: ", a - b;
	Escribir "Multiplicación: ", a * b;
	Escribir "División: ", a / b;
	Escribir "Módulo: ", a MOD b;
	Escribir "Potencia: ", a ^ 2;
FinAlgoritmo
!
Cuidado con la división por cero
Si divides entre 0, PSeInt generará un error. Siempre verifica que el divisor no sea cero antes de dividir.

4.2 · Operadores de comparación

Comparan dos valores y devuelven Verdadero o Falso. Son la base de todas las decisiones: "¿aprobó?", "¿es mayor de edad?", "¿tiene saldo?".

Operador Significado Ejemplo Resultado
= ¿Es igual? 3 = 3 Verdadero
<> ¿Es diferente? 3 <> 5 Verdadero
> ¿Es mayor? 10 > 5 Verdadero
< ¿Es menor? 2 < 5 Verdadero
>= ¿Es mayor o igual? 5 >= 5 Verdadero
<= ¿Es menor o igual? 0 <= 0 Verdadero
Ejemplo: comparar valores
Algoritmo Comparaciones
	Definir nota Como Real;
	nota <- 7.5;

	Escribir "¿Aprobó (nota >= 6)? ", nota >= 6;
	Escribir "¿Nota perfecta (= 10)? ", nota = 10;
	Escribir "¿Reprobó (< 6)? ", nota < 6;
FinAlgoritmo

4.3 · Operadores lógicos

Combinan varias condiciones en una sola. Piensa en ellos como conectores: "si tiene llave Y la puerta está cerrada", "si está vivo O tiene revive".

Operador Significado Se cumple cuando…
Y Y (ambas deben ser verdaderas) Las dos condiciones son Verdadero
O O (basta una verdadera) Al menos una condición es Verdadero
NO NO (invierte el valor) Convierte Verdadero en Falso y viceversa
Ejemplo: combinar condiciones
Algoritmo OperadoresLogicos
	Definir tiene_llave Como Logico;
	Definir nivel Como Entero;
	tiene_llave <- Verdadero;
	nivel <- 7;

	// Y: ambas deben cumplirse
	Si tiene_llave Y nivel >= 5 Entonces
		Escribir "Puedes abrir la puerta";
	FinSi

	// O: basta una
	Definir vidas Como Entero;
	Definir tiene_revive Como Logico;
	vidas <- 0;
	tiene_revive <- Verdadero;

	Si vidas > 0 O tiene_revive Entonces
		Escribir "Sigue en el juego";
	FinSi

	// NO: invierte
	Definir esta_congelado Como Logico;
	esta_congelado <- Falso;

	Si NO esta_congelado Entonces
		Escribir "Puede moverse";
	FinSi
FinAlgoritmo
✓
Tabla de verdad rápida
Verdadero Y Verdadero → Verdadero · Verdadero Y Falso → Falso · Falso Y Falso → Falso
Verdadero O Verdadero → Verdadero · Verdadero O Falso → Verdadero · Falso O Falso → Falso
NO Verdadero → Falso · NO Falso → Verdadero
Error MUY común: <- vs = típico

Este es el error más frecuente en principiantes:

<- → Asignar: guarda un valor. "Pon esto aquí".
= → Comparar: pregunta si dos cosas son iguales. "¿Son iguales?"

✅ Asignar (guardar)

vidas <- 3;       // Guardar 3 en vidas
vidas <- vidas - 1;  // Ahora vale 2

✅ Comparar (preguntar)

Si vidas = 0 Entonces  // ¿Vidas es igual a 0?
	Escribir "Game Over";
FinSi

  1. Declara: ataque <- 25, defensa <- 10, es_critico <- Verdadero
  2. Calcula dano <- ataque - defensa
  3. Si es_critico es Verdadero, multiplica el daño por 2: dano <- dano * 2
  4. Muestra si el daño es mayor a 20
  5. Muestra si puede atacar: dano > 0 Y NO esta_congelado

4.4 · Concatenar textos (unir cadenas)

Puedes unir textos con + o simplemente usar comas en Escribir. Si necesitas unir texto con un número, usa ConvertirATexto().

Unir textos y mostrar datos
Algoritmo ConcatenarTextos
	Definir nombre Como Cadena;
	Definir vidas Como Entero;
	nombre <- "Carlos";
	vidas <- 3;

	// Opción 1: comas en Escribir (más fácil)
	Escribir "Hola ", nombre, ", tienes ", vidas, " vidas";

	// Opción 2: concatenación con +
	Escribir "Jugador: " + nombre;
	Escribir "Vidas: " + ConvertirATexto(vidas);
FinAlgoritmo

5) Condicionales (Si / Según)

Los condicionales permiten tomar decisiones. El programa evalúa una condición y ejecuta un bloque u otro.

Si / SiNo / FinSi

Estructura básica
Algoritmo CondicionalBasico
	Definir vidas Como Entero;
	vidas <- 1;

	Si vidas > 1 Entonces
		Escribir "Estás bien";
	SiNo
		Si vidas = 1 Entonces
			Escribir "¡Cuidado! Última vida";
		SiNo
			Escribir "Game Over";
		FinSi
	FinSi
FinAlgoritmo

Segun / FinSegun (selección múltiple)

Cuando hay muchas opciones fijas
Algoritmo CondicionalSegun
	Definir dia Como Entero;
	Escribir "Ingresa un día (1-7):";
	Leer dia;

	Segun dia Hacer
		1:
			Escribir "Lunes";
		2:
			Escribir "Martes";
		3:
			Escribir "Miércoles";
		4:
			Escribir "Jueves";
		5:
			Escribir "Viernes";

6) Ciclos (Para / Mientras / Repetir)

Un ciclo repite un bloque de código. PSeInt tiene tres tipos de ciclos, cada uno ideal para una situación diferente.

Para (cantidad conocida)

Contar del 1 al 5
Algoritmo CicloPara
	Definir i Como Entero;

	Para i <- 1 Hasta 5 Con Paso 1 Hacer
		Escribir "Número: ", i;
	FinPara
FinAlgoritmo

Mientras (condición al inicio)

Repetir mientras se cumpla
Algoritmo CicloMientras
	Definir cont Como Entero;
	cont <- 0;

	Mientras cont < 3 Hacer
		Escribir "Intento: ", cont;
		cont <- cont + 1;
	FinMientras
FinAlgoritmo

Repetir (al menos una vez)

Ejecuta y luego verifica
Algoritmo CicloRepetir
	Definir num Como Entero;

	Repetir
		Escribir "Ingresa un número positivo:";
		Leer num;
	Hasta Que num > 0

	Escribir "Ingresaste: ", num;
FinAlgoritmo
✓
¿Cuál uso?
Para: cuando sabes cuántas veces repetir (ej: del 1 al 10).
Mientras: cuando depende de una condición que puede no cumplirse nunca (puede ejecutar 0 veces).
Repetir: cuando necesitas que se ejecute al menos una vez (ideal para validar datos del usuario).

Para con paso diferente

Cuenta regresiva y pares
Algoritmo PasosDiferentes
	Definir i Como Entero;

	// Cuenta regresiva: de 5 a 1
	Para i <- 5 Hasta 1 Con Paso -1 Hacer
		Escribir i;
	FinPara
	Escribir "¡Despegue!";

	// Solo pares: de 2 a 10 de 2 en 2
	Para i <- 2 Hasta 10 Con Paso 2 Hacer
		Escribir "Par: ", i;
	FinPara
FinAlgoritmo

Ejemplo: combate hasta ganar

Mientras con condición de combate
Algoritmo Combate
	Definir vida_enemigo, dano Como Entero;
	Definir golpe Como Entero;
	vida_enemigo <- 100;
	dano <- 30;
	golpe <- 0;

	Mientras vida_enemigo > 0 Hacer
		golpe <- golpe + 1;
		vida_enemigo <- vida_enemigo - dano;
		Escribir "Golpe ", golpe, ": vida restante = ", vida_enemigo;
	FinMientras

	Escribir "Enemigo derrotado en ", golpe, " golpes";
FinAlgoritmo
Error común: ciclo infinito peligro
Un Mientras puede volverse infinito si la condición nunca cambia.

❌ Infinito

Definir x Como Entero;
x <- 0;
Mientras x < 5 Hacer
	Escribir x;
	// falta: x <- x + 1;
FinMientras

✅ Correcto

Definir x Como Entero;
x <- 0;
Mientras x < 5 Hacer
	Escribir x;
	x <- x + 1;
FinMientras

7) SubProcesos y Funciones

Un SubProceso (o Función) es un bloque de código con nombre, diseñado para hacer una tarea específica. Te ayuda a no repetir código y a organizar tu lógica en partes reutilizables.

SubProceso (sin retorno)

Ejecuta acciones pero no devuelve un valor.

Definir y llamar
SubProceso Saludar(nombre)
	Escribir "¡Hola ", nombre, "!";
FinSubProceso

Algoritmo Principal
	Saludar("Mario");
	Saludar("Luisa");
	Saludar("Carlos");
FinAlgoritmo

Función (con retorno)

Calcula algo y devuelve un resultado que puedes guardar.

Función que retorna un valor
Funcion resultado <- Sumar(a, b)
	resultado <- a + b;
FinFuncion

Algoritmo Principal
	Definir total Como Entero;
	total <- Sumar(3, 5);
	Escribir "3 + 5 = ", total;

	Escribir "10 + 20 = ", Sumar(10, 20);
FinAlgoritmo

Función con lógica interna

Calcular daño con defensa y crítico
Funcion d <- CalcularDano(ataque, defensa, es_critico)
	d <- ataque - defensa;
	Si d < 0 Entonces
		d <- 0;
	FinSi
	Si es_critico Entonces
		d <- d * 2;
	FinSi
FinFuncion

Algoritmo Principal
	Escribir "Normal: ", CalcularDano(25, 10, Falso);
	Escribir "Crítico: ", CalcularDano(25, 10, Verdadero);
	Escribir "Sin daño: ", CalcularDano(5, 20, Falso);
FinAlgoritmo

SubProceso que modifica datos

Pasar parámetros por referencia
SubProceso MostrarResumen(nombre, vida, ataque)
	Escribir "=== Resumen ===";
	Escribir "Nombre: ", nombre;
	Escribir "Vida: ", vida;
	Escribir "Ataque: ", ataque;
FinSubProceso

Algoritmo Principal
	MostrarResumen("Guerrero", 100, 25);
	MostrarResumen("Mago", 60, 40);
FinAlgoritmo
!

Usa Función cuando necesitas que devuelva un valor (un cálculo, una verificación).
Usa SubProceso cuando solo necesitas ejecutar acciones (mostrar algo, modificar datos).

Crear y llenar un arreglo

	Escribir "Último: ", enemigos[3];

Mostrar todos los elementos

1. Lee el mensaje (¿qué esperaba PSeInt?).

3. Busca patrones: falta Entonces, FinSi, comillas, <-.
4. Corrige y vuelve a ejecutar (uno a la vez).

12) Buenas prácticas (hábitos que evitan bugs)

Las buenas prácticas ayudan a que el código sea fácil de leer, fácil de corregir y fácil de hacer crecer.

Estilo y claridad

✅ Nombres descriptivos: velocidad_jugador
✅ Comentarios cortos solo donde hace falta
✅ Separar lógica en SubProcesos/Funciones
✅ Evitar "números mágicos": usar variables con nombre claro
✅ Indentar correctamente cada bloque

Estructura y organización

✅ Declarar todas las variables al inicio del algoritmo
✅ Usar Repetir/Hasta Que para validar datos del usuario
✅ Probar con valores extremos (0, negativos, máximo)
✅ Dividir problemas grandes en funciones pequeñas
✅ Nombrar el algoritmo de forma descriptiva

Ejemplo: código limpio vs desordenado comparación

❌ Desordenado

Algoritmo x
Definir a Como Entero;
a <- 100;
Definir b Como Entero;
b <- 15;

Escribir "si";
FinSi
FinAlgoritmo
